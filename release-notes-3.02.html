Release, 3.02

Nov 13 2015

Logscape 3.02 is a critical patch, implementing a fix to performance issues that were being seen in some environments. It is reccomended all users upgrade to 3.02 at their earliest convenience. Below is a list of issues fixed in 3.02.

Additions in Logscape 3.02

Single Value Panel

This new widget is designed to give the user more control over what is displayed, it exposes a section where the users can freely add Javascript which the widget will then execute.

Performance Additions

This release addresses a performance issue and changes part of the execution path. It is using a new technology to the stack called OpenHFT Chronicle [Apache 2]. The issue was around event streaming of raw data, there are 3 stages, the Indexer, AggSpace and Dashboard. In each case the default implementation is to use chronicle.

To change to different impls: Dashboard, -Devent.db=chronicle, mapdb, native

Indexer & AggSpace -Devent.agg.type=chronicle,mapdb,native,chronicle.queue


To configure: chroniclemap events stored on dashboard [default]:

db.replay.event.store=1000000

-- this is the hot data map used to store search events that users interactivly filter, sort etc. It is shared between all users


chroniclemap event queue on replays streamed from IndexStores->AggSpace [default[

replay.agg.events=1000000

-- per process shared queue


Fixes in Logscape 3.02

Reshipping of Log files

Fixed an issue whereby an indexer would not acknowledge having recieved a file resulting in the forwarder continually ship the file resulting in exceptionally large files.

Re-Indexing can no longer cause IDX files to bloat

In edge cases re-indexes were not properly removing previous IDX files, instead appending to them.

Persistit Corruption

When Logscape was deployed on Windows machines it was possible for file locks to remain on persistit files, preventing the index being rebuilt.

Integer Parse exceptions

Multiline events were not being properly stored within the IDX filesystem resulting in Integer parse exceptions.
