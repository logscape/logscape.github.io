    Logscape

    By default LogScape uses the hosts OS Timezone, this however can be overwritten using the boot.properties file.

    -Duser.timezone=EST

    The process that Logscape follows is the following,

    1) Forwarder reads its timezone from OS or boot.properties

    2) Each Forwarded message contains the timezone information and is sent to the host/filename hash that maps to a Logserver instance

    3) If the logserver has no record of this it will be written to file in the /logscape_directory/work/Logserver_SERVER_/host_name/datasource.properties file, in the following format

    source.timezone=""

    Troubleshooting

    If you are having issues with an incorrect time zone,

    1) Delete datasource.properties file from /logscape_directory/work/Logserver_SERVER_/host_name directory.

    2) Stop the forwarder you are having issues with

    3) Edit the boot.properties file on the forwarder host.

    4) Restart the forwarder.
