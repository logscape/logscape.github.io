Data Source Offsets

Certain log-data such as IIS or Squid-Proxy logs will assume a TimeZone of GMT-0. By default LogScape uses the hosts TimeZone which results in the data being show with an incorrect time window. To overcome this problem it is possible to override a specific data-sources timezone with an explicit setting.

Data Source times can be overrided by using a datasource.properties file. The content of the file has the following format

$DATASOURCE.timezone=$TIMEZONE

An example entry for a data source called IIS will look like this

iis.timezone=GMT
