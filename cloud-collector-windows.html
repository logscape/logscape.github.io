Stream your Data/ Windows Server

Retrieving Windows OS Metrics

The collector is able to aggregate and transfer a wide variety of data, in this example we'll be shipping Windows OS metrics. The collector is incredibly fast to set up and requires essentially no configuration by the user.


1) Download and install the Windows Collector

The collector is available in two different versions, the Windows specific collector, or the platform independent collector, we'll be using the Windows specific collector.

Windows Collector

Once downloaded, run the installer and choose your installation directory.

2) Configure the Collector

Navigate to and open the service.json file, it can be found in -

%COLLECTOR_HOME%/etc/service.json

By default you will see -

{   "manager":"collector.logscape.com:9991" 
,"token":"[YOUR_SECURITY_TOKEN_HERE]"
,"apps":["windowsapp"]
}

First you will need to insert your Logscape security token into the token field, then, this config will send data gathered by the Unix App to the Logscape Cloud, using the security token provided. There is no need to download the apps yourself, the collector will handle the downloading and installation automatically at run time.

3) Start the collector

Once configured, you will have to start the installed service through the windows task manager, by default this is named, LogscapeCloudAgent however this can be changed in the jsl.ini should you wish.

4) Use the Logscape Windows OS Metrics workspace

Your data will shortly enter the Logscape Cloud, you can use the Windows OS metrics workspace to interpret your log data.
