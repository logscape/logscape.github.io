Users

In Logscape you can create users and add them to a data group. A data group applies allows or restricts access to data stored in the system. A data group also provides a namespace for all searches and workspaces created by users of a group. This ensures that no other users can access a Workspace belonging to another group.

See the user config page for more details.

Global and Group Workspaces

When a user logs into the system they will redirected to the custom Home Workspace for their group. The custom Workspace is an overview a groups data that users can update with charts useful to their group. When a user edits a Workspace or Search it is saved in the groups namespace, e.g if a user belong to the Dev group the search.System Health will be saved as:

dev.System Health

All searches and workspaces from Logscape Apps or that come with the default installation are in the global namespace and can be seen by all users.

Organizing Your Data



A Data Group defines a user's access to data. With data groups the administrator has granular control over what data a user can see when they log into the system.
