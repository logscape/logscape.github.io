A host filter restricts a data sources access to a predefined list of hosts. You can assign a comma separated list of hosts to include or exclude. Here a few examples

A list of hosts

This pattern will match all hosts in the environment that match start with dev1-grid, dev1-cache or dev2

dev1-grid,dev1-cache,dev2
Exclude a few hosts

Use the !host syntax to exclude hosts. The example below includes all hosts beginning with dev1 and excludes all dev2 hosts.

dev1,!dev2
using a resource group

The resource group unixes defines all hosts that run OS X,Linux or Solaris.

group:unixes

Resource Groups

A resource group is a list of agents defined by the query selector on the Agents page. Navigate to the Configure/Agents page as a Systems Administrator.

  1. Type in your query and test it by clicking the [FILTER] button
  2. Create Resource Group - once you are happy that your query is selecting the correct hosts click on the [NEW] button on the below the query option. Update the settings then click save.
  3. Your resource group should now appear in the drop down menu

Once you have created the resource group you can return to the Data Sources page and update your tag.
