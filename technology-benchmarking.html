    Benchmarks

    This page is a collection of search performance benchmarks on low spec machines. These results will provide a reference for search performance when using similar or superior hardware configurations. The data sets used will range from typical network data to application log data like log4j. The figures below are low end servers with a core density of 4 to 6 cores.

    On more modern hardware expect to see events of 1M eps or more.

    Disk Benchmarking

    A healthy Logscape environment should be CPU bound rather than IO bound. For that reason it's important to test the IO speed of your disks, to do this we're going to use a tool called iozone. IOzone is fully configurable and used for performing disk benchmarking, allowing for a variety of test formats with fully customizable datasets.

    IOzone can be downloaded for free here.

    Once IOzone has been downloaded and installed it can be ran with the following arguements to give insight into how the disk is currently performing

    iozone -T -t16 -r64k -S20480 -l -i 0 -i 1 -i 2

    Once all the tests have been complete you will have a full run down of operations, as well as the speed at which this was performed.

    Device Expected Speed
    Hard Drive 30mb/sec
    SSD 300mb/sec
    RAID-5 1.8Gb/sec

    SAN Benchmarking

    Using a SAN will requires sufficient I/O performance in order to saturate the CPU's. To benchmark a SAN the goals of the CPU performance should be understood. For example a set of 10 48 core IndexStores will require 10 x 2GB/s sustained throughput (20GB/sec). This may exceed to performance for many SAN or VM environments.

    For benchmarking your SAN device, there are various tools available. Windows: CrystalDiskMark or Linux:Iozone.

    Crystal Bench Mark will produce, read and write a series of random files which can be configured via the user interface.

    Benchmarks will vary on a case by case basis however below are some base figures as to what to expect.

    Device Expected Speed
    iSCSI 1x1gb 90mb/sec
    iSCSI 2x1gb 150mb/sec
    iSCSI 1x8GFC FCP 750mb/sec
    iSCSI 1x10gb 800mb/sec
    iSCSI 1x16GFC FCP 1500mb/sec

    The configuration for CrystalDiskMark is as follows -

    5 - the number of test passes to run.

    4000MB - The test file size, using a larger file size reduces the chance of obtaining cached data which will artificially bloat the results.

    E: - The drive to test NOTE: make sure the drive contains enough space for your test data.

    After you have configured the tool, press "All" to begin. Below is an explanation of each result.

    • Seq - Long sequential operations
    • 512k - Large operations
    • 4k - Small operations - 1 at a time
    • 4k QD32 - Small operations - many at a time

    When searching Logscape you will buffer log lines to be transferred, as such 'Seq' is the most relevant value.

    Iozone is a simple command line utility that will work on UNIX and Windows operating system, or reccomended command line is as follows.

    iozone -t 32 -s 2G -r 8k -r 16k
