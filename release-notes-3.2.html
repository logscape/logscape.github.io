Release, 3.2

18 April 2016

New Feature - Explorer


The purpose of the Explorer view is to allow the user to easily browse the file system of every agent in their environments, inspect the files by hand, as well as tail any live additions to the file.

Full documentation on the Explorer view can be found on the Logscape Support Site.

JSON Multievent support


JSON Support has been extended to include support for full json arrays, systems which generate multiple events and log this to one large json objects are now fully supported via the Logscpae JSON syntax.

A guide for use of the new extended JSON functionality can be found on the

Logscape Supportsite.

Dashboard Reload Loop Fixed


Fixed an existing issue whereby an invalid session could cause the dashboard to enter a refresh loop preventing user interaction.

Failover Iteration One


Re-added the functionality for Failovers within an environment. In Iteration one, the environment will fall back to a failover node when the manager becomes available, but will require manual intervention in order to swap back to the main manager when service is renewed. Interation two will see the introduction of failback once the manager is viewed as being stable.

In order to add failover to your environment -

  • Introduce a failover agent to your environment in the appropriate zone.
  • Deploy a connection.properties file to your environment which contains
    • manager.address=""
    • failover.address=""

Performance and Stability Improvements


A number of small stability and performance issues have been rectified in the continueing effort to make Logscape as stable and as efficient as possible.
