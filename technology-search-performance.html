Search Performance

Search performance is largely dependant on the hardware used. See the sizing guide to learn how to size your Logscape components.

In Logscape there 2 types of searches.

  • Keyword Search
  • Analytics based Search

- A keyword search looks for the keyword in the current time window. Key word searches are fast because analytics are not executed on the events as they are search.

- Analytics Based SearchIs any search that extracts data from an event and then executes the analytic described in the search. Analytics style search require for the field value to be evauluated before they are executed. Unsuprisingly because of the extra processing analytics based searches will be slower than keyword searches.

You can speed up a search by adding a keyword filter. Here's an example of two search which produce the same results

404 | _type.equals(web) url.count()
and
| _type.equals(web) url.count() responseCode.equals(404)

The first search will filter all events for the keyword '404' 404 before evaluating any fields. In this case url.count(). In the second search every web access log event is scanned, the url and responseCode fields are evaluated for every single line. Any events that do not match the search criteria are thrown away.

The first search is more efficient since events can be discarded by keyword before doing any further processing.
