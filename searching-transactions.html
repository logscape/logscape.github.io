    Transactions (txn function)

    A transaction can be traced through your data by a unique identifier(uid). The transaction is traced through the logs produced by different modules and components of your system, giving you an end-to-end view. Logscape provides the txn function which plots the durations of a transaction trace and counts the volume of concurrent transactions.

    UID.txn(PARAMETER)
    • UID - Any field that can be used as a unique identifier in your system. An example is a user session identified by it's client ip address.
    • PARAMETER - The parameter can be set to elapsed or volume. The elapsed, will give you the duration of a transaction. The volume parameter, will count the number of concurrent transactions.

    Example - In the sample data below our uid is the field user. Each user has an event associated with an event: start,calc, and end. As the system processes requests the user id is traced through the jobdispatcher and calcengine components.

    jobdispatcher.log

    2014-11-24 11:00:00	user=usr_100 event=start  val=67.343
2014-11-24 11:10:00	user=usr_101 event=start  val=320 
2014-11-24 11:13:00	user=usr_102 event=start  val=600
    calcengine.log
    2014-11-24 11:13:00	user=usr_102 event=calc  val=600
2014-11-24 11:13:00	user=usr_102 event=calc  val=321
2014-11-24 11:17:00	user=usr_102 event=end  val=775
2014-11-24 12:00:00	user=usr_101 event=end  val=340
2014-11-24 12:20:00	user=usr_100 event=end  val=212
2014-11-24 12:30:00	user=usr_100 event=end  val=212

    The elapsed parameter is used in a search expression to find the duration of each transaction in the system

    | user.txn(elapsed) chart(line-connect)
    The volume option gives a different view of the transactions in a system. It provides a count of the concurrent transactions being traced. This can be used to show users that have successfully been processed from end to end.
