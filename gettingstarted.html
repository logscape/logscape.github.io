    3.1) Import SYSLOG Data

    Syslog is a log standard that is supported by several network devices, applications and flavours of Unix/Linux operating systems. Syslog collectors are also available for Windows operating systems. Logscape runs it's own Syslog Server providing a unified approach to collecting log events.

    To enable SYSLOG to appear in Logscape, we need to do couple of changes to the configuration files and restart the service again. It is that simple.

    Note: Make sure your favourite syslog forwarder agent installed

    Changing SYSLOG Network settings

    In boot.properties file, where you have installed the recent version of Logscape add the following two properties to sysprops

    • -Dsyslog.tcp.port=1468
    • -Dsyslog.udp.port=1514

    Note: That using ports < 1024 on Unix based systems requires root privileges. Using authbind and iptables can be used to overcome this.

    After a search executes, the raw events are displayed in a table below the charts. The log events table provides a
    • Raw View - which displays the log data as it appears. There is a link with an icon below each event which will open the file that contributed the line.
    • Data Type View - This shows the log events in tabular structure determined by a data type mapping
    • CSV - Exports data to CSV format
    • Filter - filtering based on the input

    5) Adding Search to Workspace

    A Workspace is a collection of related searches providing a unique view of data being collected. The Workspaces uses a flexible grid layout allowing searches, replay events, images and videos to be placed alongside each other in different orientations.

    The default workspace is also the Home page. The Home page has the following options:

    -- edit the current workspace

    -- open from a list of workspace

    -- print the current workspace
