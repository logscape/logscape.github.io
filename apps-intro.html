The Bundle - Is an xml document that contains a description of what services a Logscape App will execute. In each service description there is a schedule parameter which controls when the Apps service is executed. This example describes a service that executes a script every 60 seconds.

     <Service>
       <name>_sys_appmetrics_
       <resourceSelection>osName <contains <Linux
       <fork>true
       <background>true
       <instanceCount>1
       <pauseSeconds>20
       <resourceSelection>hostName <containsAny <battle 
       <script>_sys_appmetrics.<groovy  _filter=mysql,mysqld_safe,mysqld </script>

Config File - The config file contains all the Searches, Workspaces, types and sources that the App needs to run.

Services These are the programs or scripts that are executed by the service sections in the bundle file.

Deployment

Each App installed and deployed must follow the naming scheme

.$NAMEApp-1.0.zip
When an App is deployed, the archive is replicated to all Agents that are part of the Logscape environment. When the status of the App is changed from UINSTALLED to INSTALLED, the archive is unpacked on each Agent and stored in the following folder
$LOGSCAPE_HOME/deployed-bundle/$NAMEApp-1.0.zip
