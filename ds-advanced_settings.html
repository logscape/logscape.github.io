    Advanced Settings

    The advanced settings tab on the Data Sources configuration page provide options for system indexing, rolling logs and line break rules

    System Indexing

    There are 3 types of system indexing that can be enabled on a datasources. These are

    • Field Discovery Indexing
    • Grokit Discovery Indexing
    • System Time Field Indexing

    Field Discovery Indexing - This option controls the key-value discovery feature on a data source. Key Value pair formats that are common in logs and data exchange format like xml and web-formats like json. The key value pairs are indexed when data is imported and become searchable fields. See the Data Discovery Page for an example.

    Grokit Discovery IndexingThis option enables field extraction defined by regular expression patterns found in the grokit.config file in the $LOGSCAPE_HOME/downloads directory. Common patterns such as ip address, urls, file paths are defined in this file. See a description of pattern based extraction on Intelligent Log Analysis page

    System Time Field Indexing This controls the granularity of the time field in the raw events section, displayed on the Search Page. The default granularity of raw log events displayed in the browser is 60 seconds. To have sub-minute granularity enable this field.

    Note: That enabling this affects only how raw log events are displayed and does not interfere with the accuracy of search, charting and aggregations

    Roll Detection

    Enable this feature if your logs have an hourly, size-based or end-of-day rolling pattern.

    Line Breaking

    The line break rule defines where a single time event begins and ends in your data. By default the line break rule for the files being imported are automatically detected. The Data Source can be configured to use a specific rule. If a rule other than Default is selected then only files matching that rule will be imported.

    • Single Line -This rule is common with CSV formats where each event fits on a single line.
    • Year -Import files that start with the Year in their timestamps.
    • Month - Import files that begin with the Month in their timestamps. e.g Apr, Mar etc
    • MonthNumeric - Import files that being with the numerical month in their timestamp
    • DayNumeric - Import files that begin with the numerical day in their timestamp.

    Time Detection Rule

    The time detection option defines the time format used by each event imported by the data source. Any file that does not match this format is rejected. By default the timestamp is automatically detected. Use this option only if the time format used in your data is obscure or if you want files with a specific date format to be associated with the data source.

    For more information see Date format overrides
