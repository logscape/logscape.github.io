Data Source

Forwarder Retention Rule

The forwarder will hold data for a predefined number of days. By default this is set to indefinite. If this parameter is set, the forwarder will delete the data after N number of days. The minimum data rentention period for the forward is 2 days. If a file is deleted within the 2 day period the Index Store will retain a timestamped copy as file showing a record of unauthourized delete operation. The data is not lost.

Index Store Retention

The Index Store will apply an archiving operation after the defined number of days. The Index Store can be set to compress the data using

  • gzip compression - Slow compression, slow decompression but compresses a text file to about 10% of original size. Searches are slower on gzipped data.
  • snappy compression - Fast compression and decompression but results in larger compressed files about 20% of original file size. Provides faster search of compressed data.

The archive operation can be set to [DELETE]. Which will delete the data of the Index Store after a number of days. Care must be taken with this option since they can lead to data loss if this is set incorrectly.
