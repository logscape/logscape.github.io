    Custom Message Action

    A message based on the triggered events can be sent to the Logscape Logs. The message uses labels to create a custom log line.

    {LABEL}

    An example line may look something like this:

    {error} {filename} {all}

    and produce the following output

    2010-02-26 15:40:01,374 ERROR New I/O server worker #1-2 (report.Schedule) TivoliAlert LON-XP3 ...\C:\logscape\wrapper.log

    Other valid fields are

    • error,fatal,warn - The level of the message.
    • hostname - The originating host
    • filename - the originating file
    • all - writes all log events leading up to the alert
    • line - the line in the source file creating the trigger
